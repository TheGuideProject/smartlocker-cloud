⚙ How It Works — System Architecture

SmartLocker is a two-tier IoT system designed for marine paint inventory management & mixing assistance. The edge device (Raspberry Pi 5) handles real-time sensor interaction, touchscreen UI, and local-first data persistence. The cloud platform (FastAPI + PostgreSQL) provides centralised management, fleet-wide monitoring, and OTA update delivery.

🖥 Edge Device (Raspberry Pi 5)

PyQt6 touchscreen UI (800×480) • Multi-reader PN532 NFC (4–50 readers via USB hub, one per slot) • Arduino Nano serial bridge for weight sensor (HX711), WS2812B LED strip & buzzer • Shelf Map with drag-and-drop reader↔slot reorder (persisted across reboots) • Tag Writer for encoding NFC tags with product data • Barcode scanner fallback when RFID is down • SQLite (WAL) for local-first persistence • Runs 24/7 via systemd service

☁ Cloud Platform (FastAPI)

Admin portal (Jinja2 + HTMX) • Owner dashboard with fleet data • REST API with JWT + API key auth • PostgreSQL database • Bidirectional sync (events, inventory, commands) • OTA update engine • Hosted on Railway

💡 Challenges & Solutions — Technical Analysis

🔌 Offline-Tolerant Sync

The edge device operates in marine environments where connectivity is unreliable. All events are stored locally in SQLite first (event sourcing pattern), queued with synced=0, and batch-uploaded when connectivity returns. UUID-based deduplication ensures no data loss even if events are sent multiple times.

🚀 Fast OTA Updates

Git-based OTA with automatic rollback on failure. Commands are delivered via dual path: heartbeat response (~60s) for fast delivery + config polling (~120s) as fallback. On the edge, git pull + service restart with systemd ensures zero-downtime updates.

🛡 Crash Protection

The edge runs as a systemd service with Restart=always and StartLimitBurst=10. Health monitoring tracks CPU temperature, RAM usage, disk space, and GPU throttling. Automatic alerts when thresholds are exceeded. System recovers from crashes within seconds.

🛠 Event Deduplication

Every event has a unique UUID generated at creation time. Cloud ingestion uses idempotent processing — duplicate UUIDs are silently ignored. This prevents data corruption when events are re-sent after connectivity drops.

⚠ Poison Pill Prevention

A single corrupted event in the sync queue could block the entire pipeline indefinitely. Solution: per-event error handling with try/except, individual retry when batch fails, and automatic skipping of broken events after max retries. The sync queue never gets stuck.

📈 Real-Time Monitoring

Heartbeat every 60 seconds carrying system telemetry: CPU temperature, RAM %, disk usage, throttle status, pending sync count, and current software version. Cloud dashboard displays real-time device health with alert thresholds.

🎨 Color Management

Marine paints have complex color names (e.g., "Redbrown 6179", "Signal Red 3000"). A 4-step resolution algorithm matches colors: exact → strip numbers → first word → substring match. Cloud provides an RGB hex picker that syncs colors_json to edge devices, with a 30+ colour dictionary fallback.

📦 Vessel Inventory Sync

Cloud aggregates stock adjustments per vessel into a unified inventory. This data syncs to edge via the config API endpoint. The edge merges cloud inventory with live RFID physical data to provide a complete real-time stock picture on the touchscreen.

🎯 How It Will Work — Complete Vision

When fully deployed, SmartLocker is a smart paint locker installed aboard vessels. It combines RFID tracking, weight sensing, LED guidance, and cloud monitoring to automate paint inventory management — eliminating manual spreadsheets, preventing waste, and ensuring compliance with maintenance schedules.

📦 Physical Locker

A metal cabinet with 40–50 shelf slots, each holding one paint can. Under each slot sits a PN532 NFC reader connected via USB hub to a Raspberry Pi 5. An Arduino Nano drives the weight sensor, WS2812B LED strip (one LED per slot), and buzzer. An 800×480 touchscreen is mounted on the door for operator interaction. A USB barcode scanner provides fallback identification when RFID is unavailable.

🔧 Initial Setup

1. Install the locker and connect all hardware (PN532 readers, Arduino, RPi, touchscreen).
2. Power on — the system auto-detects all readers and the Arduino.
3. Pair the device with the cloud via 6-digit code.
4. Products, recipes, and maintenance charts sync automatically.
5. Use the Shelf Map REORDER screen to assign each physical reader to the correct slot position (persisted across reboots).
6. Use the Tag Writer to encode each paint can's NFC tag with product data (PPG code, batch, name, colour).

✅ Daily Operation

Stocking: Place a tagged paint can on any slot → NFC reader identifies the product → LED turns green → inventory updated locally and synced to cloud.

Taking a can: Remove a can → system detects absence → if no active mixing session, triggers unauthorized removal alarm (red LED + buzzer + 30-second barcode scan window). Crew must scan the barcode to confirm.

Returning a can: Place the can back → system measures weight difference to estimate paint consumed → inventory adjusted automatically.

Weight alarm fallback: If RFID fails entirely, the shelf weight sensor detects significant changes (>5kg sustained for 10s) and triggers the barcode scan alarm.

⚗ Guided Mixing

1. Select area: Tap "Paint Now" → select the area to paint from the maintenance chart.
2. Auto-calculate: System calculates required quantities of base, hardener, and thinner based on area × coverage rate × coats.
3. LED guidance: The correct slot LED lights up to guide the operator to the right can.
4. Live weighing: Place the mixing bucket on the scale → pour paint → real-time weight display with colour zones (blue=pouring, yellow=approaching, green=target, red=over).
5. Buzzer feedback: Tick while pouring, warning at 90%, target reached confirmation, error if overpoured.
6. Pot life timer: After mixing, countdown shows remaining working time.
7. Record: Session saved locally + synced to cloud with all measurements.

🌎 Fleet Monitoring (Cloud)

Ship owners and PPG managers access the cloud dashboard to see:
• Real-time inventory levels across all vessels
• Mixing session history with ratio compliance (in-spec / out-of-spec)
• Device health: CPU temperature, connectivity, last heartbeat
• Consumption trends and reorder alerts when stock is low
• Remote commands: restart, reboot, OTA software updates
• Maintenance chart assignment per vessel

🚀 Production Scale

Demo (current): 3–4 PN532 readers + 1 Arduino on a single RPi 5. Sequential RFID polling (~200ms per cycle).

Production: 40–50 PN532 readers via powered USB hubs. Automatic switch to parallel thread pool polling when reader count exceeds 6 (capped at 16 threads). Readers are mapped to slots via the touchscreen reorder UI — USB port enumeration is non-deterministic across reboots, so the mapping is persisted in the local database and restored on startup.

📚 Quick Guide — How to Use the System

  1. Initial Setup: Pair a new device using the 6-digit pairing code from the Pairing page. The device will automatically sync products, recipes, and inventory.
  2. Product Management: Add or edit products in the Products page. Assign colours using the RGB picker for visual identification on the edge touchscreen.
  3. Inventory Management: Add stock adjustments via the Inventory page. Stock levels sync to edge devices and display alongside RFID physical data.
  4. Mixing Operations: On the edge touchscreen, select a recipe and follow the LED-guided mixing process. Completed sessions sync to the cloud automatically.
  5. Maintenance Charts: Upload maintenance chart PDFs from the Charts page. These are referenced by vessel for scheduled paint application.
  6. Device Monitoring: Monitor device health from the Devices page. View CPU temperature, RAM, disk usage, and pending sync queue. Set up alerts for critical thresholds.
  7. OTA Updates: Push software updates from the Devices page. Updates are delivered via heartbeat (~60s delay when online). The device auto-restarts after update.
